①
Table-first layout for desktop. Traders scan the order board the way they scan a brokerage terminal — left-to-right ordertag → symbol → side → qty → price → status → broker → actions. The fixed-width columns keep eye-tracking consistent across re-renders even as rows shuffle in and out via SSE.
②
Status pill ordering reflects severity. PARTIAL (amber) sits between WORKING (green) and the terminal states because it's the one that most needs attention. Color-coding is consistent across the cockpit — the same green-amber-red mapping that /today uses for candidates.
③
Broker badge is colored per broker. Zerodha gets a kite-orange tint, AngelOne a brand-blue tint, Dhan a contrast cyan. Multi-broker is the future state; even with a single-broker OMS instance, the column matters for historical reads where brokers may differ.
④
Selected row stays highlighted with a 2px primary-color left edge and a faint tinted background. The detail drawer on the right syncs to the selection. Clicking elsewhere doesn't deselect — only an explicit close (×) on the drawer or clicking another row does.
⑤
The timeline is the audit trail in visual form. Each event has a precise IST timestamp and a one-line description. Green dot = past event (confirmed), amber dot = current/in-progress, red dot = error or terminal failure. The live tail at the bottom auto-scrolls as new events arrive via SSE.
⑥
Inline Modify and Cancel on each row save a click vs opening the drawer first. Both are admin-gated; for regular users these buttons render disabled with a tooltip ("Read-only — contact an admin"). The drawer hosts the longer-form Modify dialog with full RMS context.
⑦
Terminal orders (FILLED / CANCELLED / REJECTED) lose their action column and show "terminal" in muted text. They're kept in the today view until midnight IST, then migrate to /history. This avoids "ghost" buttons that would error if clicked.
