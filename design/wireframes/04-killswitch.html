Killswitch
— the emergency stop

A single screen with one job: halt all order activity, cancel everything in flight, and prevent new orders from being placed. The flow is intentionally deliberate — scope-aware, impact-previewed, two-step confirmed — because false-positive kills are operationally expensive but missing-a-real-kill is catastrophic.

①
Disarmed = no urgency, full information. When trading is healthy this screen is rarely the focus, so it doubles as the "kill switch readiness dashboard." Hero state pill shows we're disarmed; the rest of the screen tells you what would happen if you armed — so the action is never blind.
②
The button is large and round on purpose. Fitts's Law: a kill action must be hard to miss but the gesture itself (hold + type) protects against accidental clicks. The radial gradient + glow signals "important and dangerous" without being noisy.
③
Scope before action. Three options, "All brokers" as default. Each option shows its own impact line in amber so the trader can compare blast radius before committing. Single-symbol kills are surgical — useful when one position is misbehaving but others are fine.
④
Impact preview updates live. As candidates and orders change in the background via SSE, these numbers re-render. The trader always sees the cost of arming right now, not at some stale earlier moment.
⑤
Audit log is on-screen, not buried in admin. Showing the last 7 days of kill events normalizes the action — it's been used, here's when and why. Each row pairs an ARMED event with its DISARMED counterpart so context is contiguous.
⑥
Right rail tracks exposure and connectivity. When the kill button is at the center, the trader's natural next question is "do I need to use it?" — current exposure (long/short notional, net delta) and broker health are the answers.
⑦
"Last kill drill" reminds operators to test the path. We'll wire up a Monday 8 AM IST scheduled drill: arm scope=DRY_RUN, verify telemetry, disarm. If the drill hasn't run in >7 days, this card turns amber.

⑧
Armed state changes the entire chrome. Red banner at top, pulsing top edge, "SYSTEM HALTED" replacing the latency indicator, killswitch sidebar item highlighted red. Same screen, dramatically different mood — there's no ambiguity about the state. Every other route in the cockpit also surfaces the red banner.
⑨
Reason capture is mandatory for both arm and disarm. Free-text but required. This appears in the audit log and shows up in every future post-mortem. Stops "why did we kill?" / "no one remembers" from ever being an honest answer.
⑩
Disarm is super-admin only. Admin can arm — anyone with floor authority can stop trading. Disarm requires senior approval because the social pressure to "just turn it back on" mid-incident is exactly what we want to slow down. The disarm button greys out for non-super-admin sessions; the note explains why.
⑪
Live timeline of this kill event reads like a system journal. Every cancellation, skip, and state change attributed to the kill is captured with a precise timestamp. After disarm this log becomes immutable and is referenced from /admin/audit.
