Today

Orders

Live order board with status pills, broker badges, partial-fill counters. Selecting a row opens a timeline drawer with the full audit trail. Desktop + mobile.

Open → WIREFRAME 04 · /app/killswitch

Killswitch

The emergency stop. Disarmed state (ready), armed state (halted), plus the mobile confirmation modal with hold-to-arm + typed token + reason capture. Three deliberate friction points; not a single mis-click can fire it.

Open questions to resolve before code lands

  • Q1 Auth provider — Clerk vs Auth0 vs self-rolled JWT against a new auth-service. Internal-only means we can keep this simple, but role hierarchy needs to land somewhere.
  • Q2 BFF or direct? The cockpit speaks to OMS + derivatives + a new RMS engine. A BFF is cleaner but +1 service to operate. Direct calls are faster to ship but coupling grows.
  • Q3 SSE host — does the BFF own the stream, or does OMS push direct? Affects how auth + reconnect work.
  • Q4 "AUTO_THRESHOLD" — who edits it? Super admin via /rms is the design. But the value is consequential (₹50k default) — confirm before Round 2.
  • Q5 Manual signal entry — Rakesh currently posts 5 stocks to Telegram. Does /today get a "+ Add manual candidate" button, or does the Telegram bot pipe into the same queue server-side?
